# REVI · Планировщик «Завоевание каньона»

Веб-приложение для расстановки игроков альянса **REVI** по трём линиям ивента
**«Завоевание каньона»** против соперника (например **BDSM**).

## Возможности

- Загрузка ростера CSV / Excel / вставка текста (столбцы **ник** + **мощь**)
- Ввод состава соперника по линиям (файл со столбцом `lane`/`линия` или вручную)
- Стратегии:
  - Баланс
  - Зеркало
  - 2 сильные + жертва
  - Давление на слабую
  - Контр-эстафета
  - Максимум флагов (локальный поиск + симуляция)
- Drag-and-drop между линиями и скамейкой
- Симуляция эстафетного боя и прогноз флагов
- Экспорт расстановки в CSV

## Правила ивента (кратко)

По инструкциям со скринов:

- 3 линии (лево / центр / право), побеждает сторона с большим числом флагов
- Бой на линии — **эстафета**: победитель идёт дальше с остатком войск
- Порядок на линии: **от слабых к сильным**, вручную не меняется
- Лимит боёв отряда: 2 (юнцы) / 3 (элита+) — в приложении настраивается
- Макс. войск на линию зависит от ранга альянса — в приложении задаётся числом

Модель боя в MVP упрощённая:

```text
остаток = max(0, P_win − k · P_lose)
```

Коэффициент `k` настраивается (по умолчанию 1).

## Формат таблицы ростера

CSV:

```csv
nick,power
PlayerOne,18500000
PlayerTwo,16200000
```

Для соперника можно добавить линию:

```csv
nick,power,lane
EnemyA,20000000,left
EnemyB,18000000,center
EnemyC,17000000,right
```

Значения `lane`: `left`/`лево`, `center`/`центр`, `right`/`право`.

Примеры: [`public/sample-revi.csv`](public/sample-revi.csv), [`public/sample-enemy.csv`](public/sample-enemy.csv).

## Разработка

```bash
npm install
npm run dev
```

Сборка:

```bash
npm run build
npm run preview
```

## GitHub Pages

1. Создайте репозиторий и запушьте содержимое этой папки.
2. В Settings → Pages выберите источник **GitHub Actions** или ветку `gh-pages`.
3. Либо вручную:

```bash
npm run build
# содержимое dist/ — статический сайт (base: ./)
```

Workflow уже лежит в [`.github/workflows/deploy.yml`](.github/workflows/deploy.yml):
после пуша в `main` сайт собирается и публикуется на Pages.
